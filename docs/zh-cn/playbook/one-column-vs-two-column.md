# 单栏 vs. 双栏

双栏简历看起来很时尚。但数据表明，它们会让你失去面试机会。

## 招聘人员实际如何阅读简历

[TheLadders 眼动追踪研究](https://www.theladders.com/career-advice/why-do-recruiters-spend-only-7-4-seconds-on-resumes) (2012, 更新于 2018) 使用眼动追踪硬件跟踪了 30 名专业招聘人员长达 10 周的时间。关于排版的关键发现：

- 招聘人员在做出匹配/不匹配决定前，平均花费 **7.4 秒**进行初始扫描。
- 眼睛遵循 **F 模式**：从左上角开始向右扫描，然后沿着左边缘向下移动并再次向右扫描。
- 双栏排版被认为是一个**负面的设计元素**，被认为是“杂乱的”，它打断了自然的扫描流畅度。
- 为了应对这种杂乱，招聘人员将注意力集中在页面上**更少的区域**，这意味着他们阅读的内容*少于*他们在单栏简历上阅读的内容。

单栏排版自然地符合 F 模式。双栏强迫眼睛去选择遵循哪一栏，打破了扫描。

### 眼动追踪研究的注意事项

这项研究是关于招聘人员阅读行为被引用最广泛的来源，但它有真正的局限性：

- **30 名招聘人员**是一个很小的样本。当时美国大约有 39,000 家人事机构。
- 该方法论从未经过同行评审。招聘人员是如何计时的、使用了哪种眼动追踪硬件以及每次会话审查了多少份简历都没有披露。
- TheLadders 是一个招聘板，而不是一个研究机构。“7.4 秒”这个数字起到了营销内容的作用。
- 该研究没有将分栏布局变量与其他设计变量（字体选择、密度、留白）区分开来。“杂乱”的发现可能反映的是被测试的特定模板，而不是一般的双栏排版。

尽管有这些局限性，但 F 模式的发现与数十年来关于人类如何扫描文档和网页的更广泛的用户体验眼动追踪研究是一致的。

## ATS 系统如何解析分栏

“现代 ATS 能读取任何内容”的看法是错误的。虽然像 Greenhouse 和 Lever 这样的顶级系统正在改进，但大多数 ATS 市场仍在使用简单的线性文本提取。

### 解析机制

大多数 ATS 解析器**严格地从左到右、从上到下**读取文档。当他们遇到双栏排版时，他们不会先处理 A 栏再处理 B 栏。他们读取左栏的第 1 行，紧接着读取右栏的第 1 行，把不相关的内容混在一起。

如果你的左栏是“Skills”，右栏是“Experience”，ATS 的输出可能看起来像：

```
Python  Senior Software Engineer
React   Acme Corp, 2021-2024
SQL     Led migration of payment service...
```

这在招聘人员的后端仪表板中产生了无法阅读的乱码。你的简历并没有被“拒绝”，它在关键词搜索中变得**不可见**，因为解析出的字段被弄乱了。

### Enhancv ATS 解析研究 (2023)

[Enhancv](https://enhancv.com/blog/busting-ats-myths/) 将来自 Microsoft Office、Canva、Google Docs 和他们自己构建器的数十种简历模板针对 Indeed 的 ATS 进行了测试，评估了每个部分解析的准确性。

**总体平均值：**

| 排版 | 平均解析分数 |
| ---- | ------------ |
| 单栏 | 93%          |
| 双栏 | 86%          |

**双栏简历损失最大的部分：**

| 部分                    | 单栏  | 双栏 |
| ----------------------- | ----- | ---- |
| Skills                  | 65%   | 46%  |
| Education               | ~100% | 88%  |
| Certifications          | ~100% | 86%  |
| LinkedIn / social links | 100%  | 82%  |

技能解析下降了将近 20 个百分点，这是招聘人员筛选最多的部分。

**“构建良好的工具”例外**

Enhancv 还进行了一项过滤分析，仅查看 Google Docs 和他们自己的产品（排除了 Canva 和 MS Office）。在这个子集中，双栏模板的得分实际上**高于**（98%）单栏（95%）。这表明栏是如何构建的（语义结构 vs. 文本框）比栏数本身更重要。

**Enhancv 研究的注意事项**

在信任 98% 的双栏分数之前，请考虑研究的局限性：

- **小样本：** 98% 的数字来自一个经过过滤的子集，并且原文承认它仅在“某些情况下”适用。
- **单一 ATS：** 仅针对 Indeed 进行了测试。在像 Workday 或 Taleo 这样更严格的系统上，结果会有所不同。
- **总体事实：** 当包含所有工具时，**除位置标题外，单栏在每个部分的解析都更好**。
- **技术上的不透明性：** 该研究没有披露被测试的具体模板，这让人很难独立核实什么使一个模板“构建良好”。

### 特定的 ATS 平台如何处理分栏

不同的系统以不同的方式失败。[Jobscan 向 Lever 提交了一份双栏简历](https://www.jobscan.co/blog/resume-tables-columns-ats/)并发布了解析结果作为证据：ATS 成功提取了工作经验，但**完全忽略了第二栏**。申请人的技能、摘要和联系方式消失了。

虽然没有任何系统能完美地处理分栏，但各供应商的行为有所不同：

- **Taleo (Oracle)：** 严格从左到右读取。双栏导致解析器在各个部分之间随机跳转，打乱了内容（[Hireflow](https://hireflow.net/blog/taleo-resume-parsing-problems-explained)）。
- **iCIMS：** 通常会完全跳过放置在表格或分栏中的内容（[Princive](https://www.princive.com/ats/systems/icims)）。
- **Workday：** 非常严格。除非使用原生的 Microsoft Word 分栏格式（绝不要用文本框）创建，否则双栏将会失败（[Princive](https://www.princive.com/ats/systems/workday)）。
- **Greenhouse / Lever：** 比旧系统更宽容，但在复杂排版上仍不可靠（[AI ResumeGuru](https://airesume.guru/blog/single-column-vs-two-column-resume-which-layout-gets-more-interviews/)）。

### Canva 和设计工具导出

从 Canva 或 Figma 等设计工具导出的简历值得特别提及。这些工具使用绝对定位生成 PDF，将每个文本块放置在页面上准确的坐标处。解析器没有语义上的阅读顺序可遵循。行业估计对于旧的 ATS 系统，设计工具导出的解析成功率在 **5% 以下**。即使在现代系统上，结果也是不可靠的。

## 建议

**当通过在线门户网站提交申请时，我们建议使用单栏排版**。原因很直接：单栏设计将解析错误的风险降至最低，确保你的信息几乎被所有的 Applicant Tracking Systems 准确读取。

然而，**双栏排版仍有其用武之地。它们非常适合直接沟通，例如给招聘经理发电子邮件、在社交活动中分发打印的副本，或在你的个人网站上分享。**在这些场景中，双栏设计提供的视觉吸引力和对空间的有效利用可能会非常有效。
